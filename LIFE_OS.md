# Life OS — 定义文档

## 核心问题

> **Life OS 是什么？**

Life OS = **个人操作系统**：管理一个人的时间、精力、社会关系的自动化决策系统。

类比：
- macOS = 管理计算机的时间片、内存、进程、网络
- Life OS = 管理人的时间片、精力、任务、社会关系

---

## 最小可行 Life OS：三件事

**Life OS 不是**一个大而全的系统。它从三个最小场景开始：

### 场景 1：早晨决策（最高频）

**输入：**
```json
{
  "task": "今天有3件事：写报告/健身/见客户，精力80%，时间8h",
  "emotion_state": {"P": 0.3, "A": 0.5, "D": 0.6}
}
```

**输出：**
```
1. 08:30-10:30  写报告（高认知任务，精力充沛时做）
2. 10:30-11:00  休息/咖啡（缓冲）
3. 11:00-12:30  见客户（人际交互，激活状态适合）
4. 14:00-15:30  写报告（下午第二段）
5. 16:00-17:00  健身（释放压力）
```

**Life OS 决策逻辑：**
- 高认知任务 → 精力 > 70% 时做
- 人际任务 → 激活度 A > 0.3 时做（社交能量充足）
- 运动/恢复 → 精力 < 50% 或需要情绪调节时做
- 深度工作 → 愉悦度 P > 0 时做（正面情绪支持专注）

---

### 场景 2：精力-任务匹配（核心引擎）

**输入：** 精力状态 + 待办列表

```json
{
  "energy": 45,          // 0-100
  "available_hours": 3,
  "tasks": [
    {"name": "写融资BP", "cognitive_demand": 90, "deadline": "明天"},
    {"name": "回邮件", "cognitive_demand": 20, "deadline": "今天"},
    {"name": "给妈妈打电话", "social_demand": 30, "emotional_benefit": 80},
  ]
}
```

**决策：**
- 低精力 + 短时间 → 只能做邮件，不能做BP
- 精力 45% → 不适合高认知任务 → BP 推迟
- 邮件是低能耗 → 立即做
- 打电话给情感高收益 → 即使精力低也值得做

---

### 场景 3：情绪触发的时间重新分配

**输入：** 情绪状态变化

```json
{
  "before_task": "写BP",
  "emotion_state_before": {"P": 0.5, "A": 0.6, "D": 0.5},
  "emotion_state_now": {"P": -0.4, "A": 0.3, "D": -0.2},
  "cognitive_demand": 90
}
```

**输出：**
```
情绪从 兴奋(P=0.5) 变为 焦虑(P=-0.4, A=0.3)
焦虑时认知能力下降，不适合继续写BP
建议：
- 切换到低认知任务（邮件/整理文件）
- 30分钟焦虑情绪处理
- 情绪稳定后再恢复BP
```

## Life OS 三层架构

```
┌─────────────────────────────────────────────┐
│              Life OS 调度层                  │
│  schedule(task_list, emotion_state, energy) │
│  → 输出: 排序 + 时间分配 + 切换触发           │
├─────────────────────────────────────────────┤
│              Judgment 引擎                   │
│  check10d(task, emotion_state=PAD)           │
│  → 输出: 任务优先级 + 执行建议                 │
├─────────────────────────────────────────────┤
│              Emotion System                 │
│  PAD状态 → 情绪标签 → 行为调制               │
│  detect_emotion() → get_pad_state()         │
└─────────────────────────────────────────────┘
```

---

## 最小可执行任务清单

| 任务 | 描述 | 依赖 |
|------|------|------|
| `schedule()` | 输入任务列表+精力，返回排序执行计划 | emotion_adapter ✅ |
| `energy_check()` | 判断当前精力是否支持某类任务 | judgment ✅ |
| `emotion_trigger()` | 情绪变化时触发任务切换 | emotion_system ✅ |
| `life_os_run()` | 早晨决策主入口 | schedule + judgment |

---

## Life OS ≠ 什么

- **不是**日历管理（日历是输出，不是引擎）
- **不是**任务列表（待办清单是输入，不是决策）
- **不是**健康追踪（健康数据是输入）
- **不是**CRM（人际关系是上下文，不是目标）

**Life OS = 基于精力+情绪的动态任务调度**

---

## 下一步实现路径

1. **最小 CLI**：`python life_os.py "今天3件事"`
2. **Emotion 集成**：从 emotion_system 读取 PAD 状态
3. ** Judgment 集成**：用 `check10d(task, emotion_state=PAD)` 做决策
4. **闭环**：执行后记录 outcome，更新权重

> **结论**：Life OS 的最小版本就是 **每天早上问一次"今天最适合做什么"** 的自动化决策系统。不是什么大系统，是把 Judgment × Emotion × 精力管理串起来的那个调度器。
